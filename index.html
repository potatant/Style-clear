The Problem

Fast-fashion apps are
deliberately overwhelming.

Open SHEIN and you're hit with pop-ups, countdown timers, floating discount badges, and an infinitely scrolling grid that offers no natural stopping point. This isn't accidental — it's designed to manufacture urgency. But the cognitive cost is real: overwhelmed users abandon carts, make regretted purchases, and disengage entirely.

This project asks: can we reduce that overload without undermining the platform's commercial utility? The answer, grounded in Cognitive Load Theory and Choice Overload research, is yes.

01
Heuristic Audit
Evaluated the SHEIN interface against Nielsen's ten usability heuristics. Identified four recurring failure categories: unprompted pop-ups, nine competing visual elements per product card, promotional navigation that obscures product-type access, and persistently fabricated urgency cues.
Discovery
02
Mixed-Methods Survey (n = 42)
Distributed a 15-question online survey to university-age SHEIN users. Measured cognitive load, decision fatigue, cart abandonment behavior, and redesign preference after five-minute prototype exposure. Two open-ended questions captured qualitative themes.
Research
03
StyleClear Prototype + Evaluation
Built a fully interactive desktop web prototype implementing evidence-based HCI principles. Participants completed a timed task on both interfaces. Comparative evaluation measured task completion time, perceived clutter, and preference across four dimensions.
Design + Validate
Key Findings

What 42 users
actually told us.

Likert-scale data, timed task evaluation, and thematic analysis of open-ended responses produced a consistent picture: the problem is urgency design, not product volume.

Design Decisions

Every change was
theory-driven.

✕  Original Interface Problems
✕Unprompted modal pop-ups interrupt browsing flow on every entry
✕9 competing visual elements per product card (badges, prices, icons, overlays)
✕Countdown timers and "Only 3 left!" warnings manufactured regardless of actual stock
✕Filters buried behind navigation — interrupts browsing to access them
✕Infinite scroll with no defined browsing unit or stopping point
✕Promotional categories (Flash Sale, Trending) obscure product-type navigation
✓  StyleClear Solutions
✓Zero unsolicited pop-ups — promotional info moved to a static info strip
✓Max 5 elements per product card — progressive disclosure hides the rest
✓All urgency cues eliminated — sale badges retained as small, muted, single-position only
✓Persistent sidebar filter panel — always visible, never interrupts browsing
✓Pagination replaces infinite scroll — defined browsing unit per session
✓Functional categories (Women, Men, Sports) + attribute-based sidebar navigation
🧠
Cognitive Load Theory
Sweller (1988) — reducing extraneous load frees working memory for actual decision-making. Every removed element is a direct CLT intervention.
📐
Gestalt Proximity
Related product attributes grouped visually. Whitespace separates unrelated content. The eye follows intentional structure, not competitive chaos.
🔍
Progressive Disclosure
Shipping, materials, size guides — present but hidden behind expandable accordions. You access detail when you want it, not before.
⚖️
Choice Overload Research
Iyengar & Lepper (2000) — excess options create stress, not satisfaction. The redesign shows the same products with structured hierarchy, not fewer choices.
